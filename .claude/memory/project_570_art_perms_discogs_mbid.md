**Defect 2 ‚Äî Discogs id poisoned `mb_albumid`.** Beets' NATIVE discogs plugin sets `AlbumInfo.album_id`/`releasegroup_id` to NUMERIC Discogs ids; beets core `Info.MEDIA_FIELD_MAP` maps `album_id -> mb_albumid`, so every Discogs import wrote a bare int into `mb_albumid` + the `MUSICBRAINZ_ALBUMID` tag ‚Üí Jellyfin `new Guid()` FormatException. **The "plugin patch" the codebase comments reference (beets_album_op.py, beets_db.py, harness ~L531) never existed** ‚Äî likely lost in the HM‚Üímodule beets migration. 408/8407 albums were poisoned (all also carried the id in `discogs_albumid`). Fix: `_neutralize_discogs_provider_ids` in `harness/beets_harness.py::_apply_decision` (after the `_mbid_swap_event` audit) blanks a chosen Discogs candidate's `info.album_id`/`releasegroup_id` to `""`; beets then applies empty `mb_*` while `discogs_albumid` survives. Dup detection still works (beets `duplicates_query` = AndQuery over [mb_albumid, discogs_albumid]). **TRAP: beets `Info.item_data`/`raw_data` are `@cached_property`** ‚Äî mutating `album_id` after they're read is invisible unless you `info.__dict__.pop("item_data"/"raw_data")`. hasattr-gate keeps the shared `choose_item`/TrackInfo path a no-op.

**REGRESSION fixed 2026-07-09 (post-cutover):** the non-root cutover provisioned the library sub-roots (`Beets/`, `Incoming/`) but MISSED their shared parent `/mnt/virtio/Music` (which was `root:root 0755`), where the beets library DB + its SQLite rollback journal, `beets-import.log`, and `.harness-mutations.jsonl` all live. A non-root cratedigger couldn't create the journal in a 0755 root dir ‚Üí beets `DBAccessError: attempt to write a readonly database` ‚Üí every import that reached the beets-write step failed (`decision=import_failed`). LESSON: for a non-root service, the DB's PARENT DIR must be group-writable, not just the file (SQLite needs to create `-journal`/`-wal` siblings). The file being world-writable (777) is NOT enough. Fix: `chmod 2775` + `chgrp users` on `/mnt/virtio/Music`, made declarative via a wrapper tmpfiles `d /mnt/virtio/Music 2775 root users -` (applies cleanly ‚Äî parent `/mnt/virtio` is root-owned, unlike the token's cratedigger‚Üíroot transition). Blast radius: 1 request (485 The National ‚Äì Tarquin Roughs), self-healed to `wanted`. **Denylist: a hard `import_failed` exception aborts BEFORE the denylist write, so the regression denylisted NO ONE** (verified: 0 source_denylist entries with a system/import-failure reason; `source_denylist` is per-request `(request_id, username, reason)` in `lib/pipeline_db/misc.py`). Don't remove the legitimate quality/validation-reject denylists.
